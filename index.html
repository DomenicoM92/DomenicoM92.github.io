Software Reuse - Bachelor's degree thesis

This platform allows you to enter, search and execute software components through a web platform. By software component we mean any artifact of software engineering and therefore, documentation, source code, test classes etc. The platform exploits an OWL ontology to represent knowledge and semantically report it, in order to provide an approach for the storage, management and rapid recovery of a large number of software components in an industrial context. An information retrieval (IR) engine will then exploit the information stored within the ontology to improve the recovery accuracy of the components. This is because normally the files that contain the code of the software components are made up of particularly significant terms that mostly refer to the concepts of the problem you want to solve with it. The platform allows you to test a component in all its functions, to clearly and comprehensively view the content of the components, and to download any component. For now it is possible to manipulate only projects of type Java Project but it is possible to extend to any new language. For development, the following tools and technologies have been used: Node.js, Neo4j, Apache Jena, Apache Jena Fuseki, Apache Lucene, Apache Solr, Java Reflection, JDT Core.

Web App E-Commerce "Virtual Store"

The e-commerce site developed by us, for the examination of Web Programming, is aimed at those who, after a registration, want to buy / download online digital content of various kinds. The V-Store aims to provide users with multimedia content provided by various companies in the industry, offering a user-friendly platform so as to be simple and intuitive in everyday use. The V-Store site contains downloadable / purchasable content from registered users to the platform. The contents are organized in sections (app, music, etc.). The user can browse the store to search for content, view and download / purchase them. In addition, the user has a wish list in which he inserts the content he would like to buy / download. The site organizes one or more contents that the user wishes to purchase within a wish list, the user then makes a payment (through a previously registered payment method or the use of a promotional code) to obtain the products at the inside. The Administrator user uses site control / monitoring, content management and site management (graphic and structural aspects).
